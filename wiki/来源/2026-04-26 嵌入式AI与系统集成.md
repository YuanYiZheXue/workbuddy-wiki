## 二、OTA 远程烧录

### 2.1 为什么不能交给 AI 直接烧录

| 问题 | 说明 |
|------|------|
| 可靠性问题 | AI 生成的 C/C++ 代码在复杂逻辑和硬件操作上正确性不足 |
| 安全性风险 | 开放无线烧录通道是极大安全漏洞 |
| 缺乏精准控制 | AI 难以理解模糊需求背后具体需要修改哪个寄存器、驱动函数 |

### 2.2 OTA 系统架构

```

│                     OTA 完整架构        │

└────────────────────────────────────────────┘
```

### 2.3 OTA 安全流程

| 环节 | 目的 | 说明 |
|------|------|------|
| 固件版本管理 | 云端存储管理不同版本固件 | 可使用阿里云、腾讯云 IoT 等平台 |
| 安全通信 | 防止固件被窃取或篡改 | 必须使用 HTTPS 或 MQTT over TLS |
| 身份认证与授权 | 确保设备只接收合法固件 | 证书、密钥等方式互相验证 |
| 固件验签 | 确保固件完整且未被篡改 | 私钥签名，公钥验签 |
| 可靠升级与回滚 | 升级失败后设备不会"变砖" | 先写入备用区，校验后切换启动 |

### 2.4 蜂窝物联网方案

加入 4G Cat.1、NB-IoT 等蜂窝模组 + SIM 卡：
- **优势**：全球任何有蜂窝信号的地方都能远程更新和数据传输
- **应用场景**：移动车辆、偏远地区、分散的户外设备

## 四、FPGA 与 ASIC

### 4.1 FPGA vs ASIC

| 对比维度 | FPGA | ASIC |
|----------|------|------|
| 开发成本 | 低（可重复编程） | 高（掩膜成本） |
| 单位成本 | 高 | 低（大批量） |
| 性能 | 较高 | 最高 |
| 功耗 | 较高 | 低 |
| 灵活性 | 高（可重构） | 无 |
| 开发周期 | 短 | 长 |

### 4.2 ASIC 商业模式

- **IP 核授权**：ARM、RISC-V IP 供应商模式
- **Fabless**：高通、英伟达等只做设计，制造外包台积电
- **Foundry**：台积电、中芯国际等纯代工模式
- **IDM**：英特尔、三星等设计制造一体化
