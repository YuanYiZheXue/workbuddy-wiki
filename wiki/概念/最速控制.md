# 最速控制

---

## 定义

**最速控制**（Time-Optimal Control）是在最短时间内将系统从初始状态转移到目标状态的控制策略。它是**时间最优**问题，属于最优控制理论的经典问题。

---

## 核心思想

### 庞特里亚金最小值原理
对于系统：
```
dx/dt = f(x, u, t)
```
最优控制 `u*(t)` 使性能指标：
```
J = ∫_{t0}^{t1} 1 dt = t1 - t0  （最小化时间）
```
最小。

**Hamilton函数**：
```
H = 1 + λᵀ f(x, u, t)
```
最优控制使 `H` 取最小值。

---

## 相平面分析法

### 二阶系统的最速控制
以双积分器系统为例：
```
dx1/dt = x2
dx2/dt = u  （|u| ≤ 1）
```
最优控制律：`u = ±1`（Bang-Bang控制）

**切换曲线**：
```
x2 = ±√(2|x1|)  （抛物线）
```

### 开关线
- 系统状态在相平面上的轨迹
- 碰到开关线时，控制量切换符号
- 保证最短时间到达原点

## 在金融交易中的映射

### 最优执行策略
大额订单拆分执行，目标：最短时间内完成建仓/平仓，同时最小化市场冲击。

| 控制系统 | 金融执行 |
|----------|----------|
| 状态：位置、速度 | 状态：持仓量、执行速率 |
| 控制量：推力、舵偏角 | 控制量：订单发送速率 |
| 约束：控制量有界 | 约束：单笔订单上限 |
| 目标：最短时间 | 目标：最短执行时间 or 最小冲击成本 |

###  implementation
```python
# adaptive_ma.py 中的最优执行逻辑
def optimal_execution(target_size, market_depth, max_rate):
    """
    最优执行策略 - 基于最速控制原理
    target_size: 目标交易量
    market_depth: 市场深度（冲击系数）
    max_rate: 最大执行速率
    """
    # 简化模型：最小化执行时间，约束市场冲击
    if target_size / max_rate <= market_depth * 2:
        # 可以快速执行
        return max_rate
    else:
        # 需要分段执行，避免过大冲击
        return max_rate * 0.5
```

## 与其他最优控制的区别

| 类型 | 性能指标 | 应用场景 |
|------|----------|----------|
| 最速控制 | min T（时间） | 导弹拦截、紧急避险 |
| 最小能量控制 | min ∫u²dt | 航天器轨道转移 |
| 线性二次型（LQR） | min ∫(xᵀQx + uᵀRu)dt | 通用最优控制 |
| 最小冲击控制 | min ∫(dP/dt)²dt | 大额交易执行 |

## 工程实现要点

### 1. 控制量约束
实际系统控制量都有界（|u| ≤ M），最优解通常是**继电型控制**（Bang-Bang Control）。

### 2. 切换次数
对于n阶系统，最优控制最多切换n次（最多n-1次切换）。

### 3. 奇异弧
当Hamilton函数对控制量不可逆时，出现奇异弧，需要更复杂的分析。
