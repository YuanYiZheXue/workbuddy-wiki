# OTA

## 定义

OTA（Over-The-Air，空中下载）是一种通过无线通信网络对设备进行远程固件或软件更新的技术。在嵌入式系统中，OTA 允许在不物理接触设备的情况下完成固件升级。

## OTA 系统架构

```
┌────────────────────────────────────────────┐
│                     OTA 完整架构        │
├────────────────────────────────────────────┤
│                                            │
│  [主控芯片 MCU]  ←→  [Bootloader]  ←→  [通信模块]  │
│        ↓                          ↓        │
│   应用固件                    Wi-Fi/4G/5G模块│
│        ↓                          ↓        │
│  [备用区]  ←→  [固件验签]  ←→  [身份认证]  ←→  [云平台]   │
│                                            │
└────────────────────────────────────────────┘
```

## 安全流程

| 环节 | 目的 | 说明 |
|------|------|------|
| 固件版本管理 | 云端存储、管理不同版本固件 | 可使用阿里云、腾讯云 IoT 等平台 |
| 安全通信 | 防止固件被窃取或篡改 | 必须使用 HTTPS 或 MQTT over TLS |
| 身份认证与授权 | 确保设备只接收合法固件 | 证书、密钥等方式互相验证 |
| 固件验签 | 确保固件完整且未被篡改 | 私钥签名，公钥验签 |
| 可靠升级与回滚 | 升级失败后设备不会"变砖" | 先写入备用区，校验后切换启动 |

## 蜂窝物联网方案

加入 4G Cat.1、NB-IoT 等蜂窝模组 + SIM 卡：
- **优势**：全球任何有蜂窝信号的地方都能远程更新和数据传输
- **应用场景**：移动车辆、偏远地区、分散的户外设备

## AI 辅助 + OTA 的工作模式

- **AI 辅助开发**：使用 GitHub Copilot、通义灵码等 AI 编程助手，快速生成、补全或解释代码框架
- **OTA 远程部署**：经过充分测试的固件版本通过 Wi-Fi、4G/5G 部署到设备
- **AI 不能直接烧录的原因**：可靠性不足、安全风险高、缺乏精准控制
