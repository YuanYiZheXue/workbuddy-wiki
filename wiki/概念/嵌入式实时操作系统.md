# 嵌入式实时操作系统

## 定义

嵌入式实时操作系统（RTOS，Real-Time Operating System）是一种专为嵌入式系统设计的操作系统，其特点是**确定性**（Determinism）——在严格的时间约束内响应事件。

## 核心概念

| 概念 | 说明 |
|------|------|
| 任务（Task） | 独立执行单元，拥有自己的栈和优先级 |
| 调度器（Scheduler） | 决定哪个任务在何时运行 |
| 上下文切换（Context Switch） | 保存当前任务状态，恢复下一个任务状态 |
| 中断（Interrupt） | 硬件事件触发，打断当前执行流 |
| 信号量（Semaphore） | 任务间同步与互斥 |
| 消息队列（Message Queue） | 任务间通信 |

## 调度算法

| 算法 | 特点 | 适用场景 |
|------|------|----------|
| 抢占式优先级调度 | 高优先级任务可打断低优先级任务 | 大多数 RTOS（FreeRTOS、RT-Thread） |
| 时间片轮转 | 同优先级任务平均分时间片 | 公平共享 CPU |
| 速率单调调度（RMS） | 优先级与任务周期成反比 | 硬实时系统理论分析 |

## 必学 RTOS

| RTOS | 特点 | 适用场景 |
|-------|------|----------|
| FreeRTOS | 最流行、开源、轻量 | 通用嵌入式、物联网 |
| RT-Thread | 国产、组件丰富、社区活跃 | 国产替代、物联网 |
| uc/OS-II/III | 教学经典、代码规范 | 教学、工业控制 |
| Zephyr | Linux 基金会、安全性强 | 物联网、可穿戴设备 |

## 学习路径

1. C 语言（指针、结构体）
2. 微控制器原理（STM32）
3. RTOS 基本概念（任务、调度、同步）
4. FreeRTOS 实战（任务创建、通信、内存管理）
5. 内存保护（MPU）与安全检查

## 在飞控系统中的应用

- **高可靠 RTOS**：航天级系统通常运行经过 DO-178C 认证的 RTOS
- **任务分区**：飞控、导航、通信、日志记录分属不同任务
- **硬实时要求**：控制环路必须在固定周期内完成（如 1ms、5ms）
