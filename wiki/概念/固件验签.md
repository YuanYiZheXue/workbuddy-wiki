# 固件验签

## 定义

固件验签（Firmware Signature Verification）是一种确保固件完整性和真实性的安全机制。在 OTA 更新过程中，设备用公钥验证固件上的数字签名，只有验证通过才执行升级。

## 为什么需要固件验签？

| 威胁 | 说明 |
|------|------|
| 固件篡改 | 攻击者修改固件，植入后门或恶意代码 |
| 中间人攻击 | 在传输过程中替换固件 |
| 降级攻击 | 强制设备降级到存在已知漏洞的旧版本 |
| 假冒固件 | 未经授权的固件被刷入设备 |

## 验签流程

```
云端（签名端）
  1. 对固件计算哈希（SHA-256）
  2. 用私钥对哈希值签名
  3. 将固件 + 签名一起发布
      ↓
传输（HTTPS / MQTT over TLS）
      ↓
设备端（验证端）
  4. 用公钥验证签名（确认来源）
  5. 计算固件哈希，与签名中的哈希对比（确认完整）
  6. 版本号检查（防降级）
  7. 验证通过 → 写入备用区 → 切换启动
```

## 密钥管理

| 机制 | 说明 |
|------|------|
| 非对称加密 | 私钥签名、公钥验签（RSA、ECC） |
| 哈希算法 | SHA-256、SHA-3 |
| 安全存储 | 公钥烧录在设备 ROM/OTP 中，不可篡改 |
| 密钥轮换 | 定期更新密钥对，降低泄露风险 |

## 在 OTA 系统中的应用

- **安全启动（Secure Boot）**：设备启动时验证固件签名
- **增量更新**：只传输差异部分，同样需要验签
- **多镜像备份**：A/B 分区切换，验签失败自动回滚
