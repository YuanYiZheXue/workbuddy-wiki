# 上下文腐烂

（Context Rot）

## 定义

随着上下文窗口逐渐被填满，模型推理和完成任务的能力逐渐下降的现象。

> Context is a precious and scarce resource.

## Harness 层的应对策略

### Compaction（压缩）
- 当上下文窗口接近填满时的处理策略
- 智能地卸载和总结现有上下文内容
- 让 Agent 能够继续工作而不中断

### Tool Call Offloading（工具调用卸载）
- 将超过阈值的大工具输出保留头部和尾部 token
- 完整输出卸载到文件系统
- 模型在需要时再从文件系统读取

### Skills / Progressive Disclosure（渐进式披露）
- 避免启动时加载过多工具或 MCP 服务器
- 按需加载，保护模型免受上下文腐烂影响
- WorkBuddy 的 Skills 机制是此理念的具体实现

## 与 Wiki 方法的联系

Wiki 方法通过**持久化知识到文件系统**来对抗上下文腐烂：
- 不需要把所有知识都塞进上下文窗口
- 需要时从 Wiki 读取，而不是重新推导
- 相当于把"卸载到文件系统"系统化、结构化
