# 跨知识库链接规范

> 定义不同知识库之间页面链接的格式、时机和约束。

## 核心原则

### 1. 链接格式

**标准 Wikilink 格式**：`` 或 ``

**跨知识库链接格式**：
- 如果页面在当前知识库：直接 ``
- 如果页面在其他知识库：``
- 示例：
  - `[[概念/元一思想]]` — 链接到当前知识库的 `概念/元一思想.md`
  - `` — 链接到哲学知识库的 `概念/道家思想.md`
  - `` — 链接到计算机知识库的 `概念/上下文工程.md`

### 2. 何时创建跨知识库链接

**必须创建跨知识库链接的情况**：
1. **概念在不同知识库中都有出现**
   - 例如："上下文工程"在 `wiki/概念/` 和 `计算机/概念/` 中都有
   - 应该在两个页面之间创建双向链接

2. **实体跨知识库出现**
   - 例如："Geoffrey Huntley"在 `wiki/实体/` 和 `计算机/实体/` 中都有
   - 应该在两个页面之间创建双向链接

3. **概念页引用了其他知识库的实体页**
   - 例如：`wiki/概念/Agent Harness.md` 引用了 `计算机/实体/Anthropic.md`
   - 应该创建链接：``

**可以不创建跨知识库链接的情况**：
1. **概念只在一个知识库中出现**
   - 不需要创建跨知识库链接
2. **实体只在一个知识库中出现**
   - 不需要创建跨知识库链接

### 3. 链接约束

**每个页面最多 5 个跨知识库链接**：
- 避免页面被跨知识库链接淹没
- 只链接最相关、最重要的跨知识库页面

**优先链接到概念页**：
- 概念页是知识库的核心
- 实体页是辅助性的

**使用有意义的显示文本**：
- 不要：``
- 应该：``

## 实施步骤

### 步骤1：识别跨知识库链接机会

**手动识别**：
1. 读取当前知识库的所有概念页和实体页
2. 识别哪些概念/实体在其他知识库中也可能出现
3. 检查其他知识库是否有对应的页面

**自动识别**（未来）：
- 使用 AI 模型来识别跨知识库链接机会
- 基于概念/实体的名称和描述

### 步骤2：创建跨知识库链接

**双向链接**：
1. 在源页面中创建跨知识库链接
2. 在目标页面中创建反向链接

**示例**：
- `wiki/概念/Agent Harness.md` 中：
  ```
  ## 相关概念

  -  — 上下文工程在 Agent Harness 中的实践
  ```

- `计算机/概念/上下文工程.md` 中：
  ```
  ## 相关概念

  -  — Agent Harness 是上下文工程的重要实践者
  ```

### 步骤3：验证链接

**手动验证**：
1. 点击每个跨知识库链接，确保它们指向正确的页面
2. 确保双向链接都已创建

**自动验证**（未来）：
- 使用 Obsidian 的 Wikilink 验证工具
- 或使用自定义脚本

## 常见错误与修复

### 错误1：链接格式错误
- **错误**：``（使用反斜杠）
- **正确**：``（使用正斜杠）

### 错误2：显示文本无意义
- **错误**：``
- **正确**：``

### 错误3：单向链接
- **错误**：只在源页面创建链接，目标页面没有反向链接
- **正确**：确保双向链接都已创建

### 错误4：链接过多
- **错误**：一个页面有10个跨知识库链接
- **正确**：每个页面最多5个跨知识库链接，只链接最相关的
