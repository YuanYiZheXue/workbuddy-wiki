# 工具能力建设顶层设计

> 本文档是「工具分层建设策略」的顶层设计，对应 `workbuddy-wiki-schema.md` 中的「工具能力建设」章节。

## 背景

工具能力是 Wiki 构建和内容分发的基础设施。当前问题：

1. **Git/Gitee 顺，其他平台难** — Git 设计 CLI-First，其他平台无统一接口
2. **网络信息获取无先验** — 直接用 WebFetch/RAG 结果，未经验证
3. **市面工具质量参差不齐** — 需要系统验证后才纳入工具库
4. **缺失工具需要立项开发** — B 站专栏、抖音发布等市面无可用工具

## 工具分层建设策略

| 层级 | 定义 | 纳入标准 | 示例 |
|------|------|----------|------|
| **第一层：成熟工具** | 已验证、稳定可用 | 在 WorkBuddy 中能稳定调用，结果可预期 | `git`、`obsidian-cli`、`defuddle` |
| **第二层：待验证工具** | 有潜力，但未经系统测试 | 建立先验，通过测试才纳入 | `xiaohongshu-mcp`、`bilibili-api-python` |
| **第三层：缺失工具** | 市面不存在，需要立项开发 | 记录需求，排期单独立项 | B 站专栏发布、抖音发布 |

## 先验验证原则

> **核心原则**：所有从网络获取的信息（WebFetch、RAG、MCP 工具返回），使用前必须先抽样验证准确性，验证通过后才能纳入 Wiki 或作为决策依据。

### 验证流程

```
工具初次使用
  ├─ 抽样验证结果准确性（对比原始数据源）
  ├─ 验证通过 → 记录到「工具验证记录表」，纳入工具库
  └─ 验证失败 → 记录失败原因，暂不纳入

工具定期复检（每月）
  ├─ 重新验证准确性
  ├─ 仍通过 → 继续使用
  └─ 已失效 → 移出工具库
```

## 缺失工具立项清单

> 按优先级排期，后续单独立项开发。

| 需求 | 描述 | 立项建议 | 优先级 | 状态 |
|------|------|----------|----------|------|
| B 站专栏发布 | B 站未开放专栏 API，`bilibili-api-python` 不支持 | 抓包分析 B 站 Web 端专栏编辑器请求，逆向 API | 🔴 高 | 📋 待立项 |
| 小红书稳定 MCP | `xiaohongshu-mcp-server` 依赖冲突，MCP 加载失败 | 解决依赖冲突，或重写一个更稳定的 MCP 服务 | 🔴 高 | 📋 待立项 |
| 抖音 / 快手发布 | 市面无可用工具 | 抓包分析，或等官方开放能力 | 🟡 中 | 📋 待立项 |
| 平台反馈自动汲取 | 目前手动将平台评论汲取到 Wiki | 监听各平台评论，自动写入 Wiki | 🟡 中 | 📋 待立项 |
| 知乎自动发布 | 目前手动复制粘贴 | 分析知乎 Web 端发布请求，逆向 API | 🟢 低 | 📋 待立项 |
| 掘金自动发布 | 目前手动复制粘贴 | 分析掘金 Web 端发布请求，逆向 API | 🟢 低 | 📋 待立项 |

## 已验证工具速查

### Git 常用命令（PowerShell 环境）

> 注意：PowerShell 环境中 `git` 可能不在 PATH，需用完整路径：`C:\Program Files\Git\bin\git.exe`。

| 场景 | 命令 |
|------|------|
| 查看状态 | `git status --short` |
| 暂存所有变更 | `git add -A` |
| 提交 | `git commit -m "message"` |
| 推送到远程 | `git push origin main` |
| 获取远程更新 | `git fetch --all` |
| 查看远程差异 | `git log --oneline HEAD..origin/main` |
| 查看标签 | `git tag` |
| 回滚到标签 | `git checkout <tag>` |
| 查看最近提交 | `git log --oneline -5` |
| 查看远程仓库 | `git remote -v` |

### Obsidian-CLI 常用命令

| 场景 | 命令 |
|------|------|
| 读取笔记 | `obsidian note get "笔记名"` |
| 创建笔记 | `obsidian note create "笔记名"` |
| 搜索 vault | `obsidian search "关键词"` |
| 获取反向链接 | `obsidian backlinks file="页面名"` |
