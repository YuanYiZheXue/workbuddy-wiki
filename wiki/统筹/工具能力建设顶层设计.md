# 工具能力建设顶层设计

> 本文档是「工具分层建设策略」的顶层设计，对应 `workbuddy-wiki-schema.md` 中的「工具能力建设」章节。

## 背景

工具能力是 Wiki 构建和内容分发的基础设施。当前问题：

1. **Git/Gitee 顺，其他平台难** — Git 设计 CLI-First，其他平台无统一接口
2. **网络信息获取无先验** — 直接用 WebFetch/RAG 结果，未经验证
3. **市面工具质量参差不齐** — 需要系统验证后才纳入工具库
4. **缺失工具需要立项开发** — B 站专栏、抖音发布等市面无可用工具

## 工具分层建设策略

| 层级 | 定义 | 纳入标准 | 示例 |
|------|------|----------|------|
| **第一层：成熟工具** | 已验证、稳定可用 | 在 WorkBuddy 中能稳定调用，结果可预期 | `git`、`obsidian-cli`、`defuddle` |
| **第二层：待验证工具** | 有潜力，但未经系统测试 | 建立先验，通过测试才纳入 | `xiaohongshu-mcp`、`bilibili-api-python` |
| **第三层：缺失工具** | 市面不存在，需要立项开发 | 记录需求，排期单独立项 | B 站专栏发布、抖音发布 |

## 先验验证原则

> **核心原则**：所有从网络获取的信息（WebFetch、RAG、MCP 工具返回），使用前必须先抽样验证准确性，验证通过后才能纳入 Wiki 或作为决策依据。

### 验证流程

```
工具初次使用
  ├─ 抽样验证结果准确性（对比原始数据源）
  ├─ 验证通过 → 记录到「工具验证记录表」，纳入工具库
  └─ 验证失败 → 记录失败原因，暂不纳入

工具定期复检（每月）
  ├─ 重新验证准确性
  ├─ 仍通过 → 继续使用
  └─ 已失效 → 移出工具库
```

## 子空间迭代管理

> **核心原则**：子空间的内容不要合并到主仓库。

### 背景

子空间 agent（哲哲/数数/算算/金金）独立迭代，主仓库只记录他们的迭代历史，不合并他们的代码或文档。

### 目录结构

```
d:\Obsidian_KN\知识库构建\
.workbuddy\
├── memory\              # 主仓库记忆文件
│   ├── MEMORY.md
│   └── YYYY-MM-DD.md
└── iterations\          # 子空间迭代记录（新增）
    ├── 哲哲.md          # 哲哲的迭代记录
    ├── 数数.md          # 数数的迭代记录
    ├── 算算.md          # 算算的迭代记录
    └── 金金.md          # 金金的迭代记录
```

### 迭代记录内容

每个 agent 的迭代记录文件应包含：

1. **基本信息**（Agent 名、负责知识库、分支名、工作空间）
2. **提交记录**（提交哈希、提交信息、说明）
3. **迭代内容**（本次迭代的核心内容）
4. **问题清单**（遇到的问题和修复方案）
5. **主仓库决策**（是否合并、是否采纳等决策）

### 工作流程

#### 子空间 agent 提交迭代

1. 子空间 agent 在自己的分支上提交代码/文档
2. 子空间 agent 通知主仓库 agent（元一）有新的迭代
3. 元一查看子空间分支的提交记录
4. 元一将迭代内容记录到对应的 `.workbuddy/iterations/X.md` 文件

#### 主仓库审查迭代

1. 元一查看子空间的提交记录和迭代内容
2. 元一评估是否采纳子空间的迭代（全部/部分/不采纳）
3. 元一在迭代记录文件中记录决策
4. 如果采纳，元一在主仓库中实现相同的功能（而不是合并子空间的代码）

#### 子空间修复问题

1. 子空间 agent 发现自己的问题（或元一指出问题）
2. 子空间 agent 在自己的分支上修复问题
3. 子空间 agent 通知元一问题已修复
4. 元一更新迭代记录文件中的「修复进度」

### Git 常用命令（PowerShell 环境）

> 注意：PowerShell 环境中 `git` 可能不在 PATH，需用完整路径：`C:\Program Files\Git\bin\git.exe`。

| 场景 | 命令 |
|------|------|
| 查看状态 | `git status --short` |
| 暂存所有变更 | `git add -A` |
| 提交 | `git commit -m "message"` |
| 推送到远程 | `git push origin main` |
| 获取远程更新 | `git fetch --all` |
| 查看远程差异 | `git log --oneline HEAD..origin/main` |
| 查看标签 | `git tag` |
| 回滚到标签 | `git checkout <tag>` |
| 查看最近提交 | `git log --oneline -5` |
| 查看远程仓库 | `git remote -v` |

### Obsidian-CLI 常用命令

> 详细 Obsidian-CLI 高级技巧请参考：**→ [[统筹/Obsidian-CLI高级技巧速查表]]**

| 场景 | 命令 |
|------|------|
| 查看版本 | `obsidian version` |
| 打开 vault | `obsidian open vault="知识库构建"` |
| 打开当天日记 | `obsidian daily` |
| 创建笔记 | `obsidian create name="笔记名" content="内容"` |
| 搜索笔记 | `obsidian search query="关键词"` |
