# 跨知识库协作指南

> **目标**：让多个 WorkBuddy 实例协作构建不同领域的知识库，并自动建立跨知识库链接。

## 架构

```
workbuddy-wiki 仓库（同一个仓库，不同分支）
├── main              ← 主知识库（wiki/）
├── kb/philosophy    ← 哲学知识库（只递交 wiki/index.md）
├── kb/computer      ← 计算机知识库
└── kb/math          ← 数学知识库
```

**核心约束**（来自 `workbuddy-wiki-schema.md` 的「会话管理」小节）：
- WorkBuddy 历史对话最多保存 100 个
- 每个会话不要超过 ~30 轮对话
- 关键信息必须写入持久化文件

## 准备工作

### 1. 安装 Git
- 下载：https://git-scm.com/download/win
- 安装时选择"Use Git from the command line and also from 3rd-party software"

### 2. 配置 Git
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### 3. 克隆主仓库
```bash
cd d:/Obsidian_KN
git clone https://github.com/你的用户名/workbuddy-wiki.git
```

## 创建新工作空间

### 方法：运行 `setup_kb_branch.py`

```bash
cd d:\Obsidian_KN\知识库构建
python scripts/setup_kb_branch.py kb/philosophy d:/Obsidian_KN/哲学知识库
```

**脚本功能**：
1. 在主仓库中创建新分支 `kb/philosophy`
2. 删除不需要的目录（只保留 `wiki/index.md` 和 `wiki/统筹/`）
3. 提交并推送到远程
4. 切回 `main` 分支
5. 创建新工作空间目录 `d:/Obsidian_KN/哲学知识库`
6. 克隆仓库到新目录，切换到新分支
7. 创建 `.gitignore`（只递交索引文件）

**结果**：
- 新工作空间：`d:/Obsidian_KN/哲学知识库`
- 当前分支：`kb/philosophy`
- `.gitignore` 已配置

## 新工作空间工作流程

### 1. 打开 WorkBuddy
- 选择 `d:/Obsidian_KN/哲学知识库` 作为工作空间
- 启动新对话

### 2. 配置 Agent 身份
在新对话中说：
```
请帮我构建哲学知识库。

配置文件已准备好：
- workbuddy-wiki-schema.md — Wiki 方法
- SOUL.md — 你的身份是「哲学构建者」
- IDENTITY.md — 你的名字是「哲哲」

请先读取这些文件，然后开始构建。
```

### 3. 构建知识库
- 提供哲学领域的原始资料（如 PDF、文章）
- WorkBuddy 会生成 `wiki/概念/`、`wiki/实体/` 等
- **只递交 `wiki/index.md` 到 Git**

### 4. 同步元数据
```bash
cd d:/Obsidian_KN/哲学知识库
python scripts/sync_meta.py
```

**脚本功能**：
1. 获取 `main` 分支的 `wiki/统筹/` 目录（包含全局视图）
2. 提交并推送到当前分支

### 5. 识别跨知识库链接机会
```bash
cd d:/Obsidian_KN/哲学知识库
python scripts/generate_ai_prompt_cross_kb_links.py
```

**脚本功能**：
1. 读取 `wiki/统筹/全局视图.yaml`
2. 生成 AI Prompt，用于识别跨知识库链接机会
3. 输出到 `wiki/统筹/ai_prompt_识别跨库链接机会.txt`

### 6. 创建跨知识库链接
- 让 AI 读取 Prompt，识别链接机会
- 保存到 `wiki/统筹/跨知识库链接机会.yaml`
- 运行：
```bash
cd d:/Obsidian_KN/哲学知识库
python scripts/apply_cross_kb_links.py
```

### 7. 递交更新
```bash
cd d:/Obsidian_KN/哲学知识库
git add wiki/index.md wiki/统筹/ wiki/概念/ wiki/实体/
git commit -m "feat(哲学): 更新索引和跨知识库链接"
git push origin kb/philosophy
```

## 主 WorkBuddy 工作流程

### 1. 推送元数据
```bash
cd d:\Obsidian_KN\知识库构建
python scripts/push_meta.py
```

**脚本功能**：
1. 确保在 `main` 分支
2. 生成全局视图（`get_knowledge_base_index.py`）
3. 提交并推送 `wiki/统筹/` 目录

### 2. 检查分支更新
```bash
cd d:\Obsidian_KN\知识库构建
python scripts/check_kb_updates.py
```

**脚本功能**：
1. Fetch 所有远程分支
2. 检查 `kb/` 分支是否有新提交
3. 输出更新状态到 `wiki/统筹/分支更新状态.yaml`

### 3. 合并分支更新
```bash
cd d:\Obsidian_KN\知识库构建
python scripts/merge_cross_kb_links.py kb/philosophy
```

**脚本功能**：
1. Fetch 指定分支的最新代码
2. 检查是否有更新
3. 合并更新到 `main` 分支
4. 推送到远程

**或者合并所有分支**：
```bash
cd d:\Obsidian_KN\知识库构建
python scripts/merge_cross_kb_links.py
```

## 自动化建议

### 主 WorkBuddy 侧
使用 cron（Linux/macOS）或 Task Scheduler（Windows）定时运行：
```bash
# 每小时检查一次分支更新
0 * * * * cd d:\Obsidian_KN\知识库构建 && python scripts/check_kb_updates.py

# 每天推送一次元数据
0 2 * * * cd d:\Obsidian_KN\知识库构建 && python scripts/push_meta.py
```

### 新工作空间侧
在 WorkBuddy 对话中，每次构建完成后运行：
```bash
python scripts/sync_meta.py
python scripts/generate_ai_prompt_cross_kb_links.py
```

## 故障排除

### 1. 合并冲突
- 手动解决冲突：`git status` 查看冲突文件
- 编辑文件，保留需要的内容
- `git add .` 然后 `git commit`

### 2. 推送被拒绝
- 先拉取最新代码：`git pull origin <branch>`
- 解决冲突后再推送

### 3. 脚本运行失败
- 检查是否在正确的目录中
- 检查 Git 是否已安装并配置
- 检查远程仓库地址是否正确
