# 新对话启动模板

> **使用方法**：每次开启新对话时，复制以下模板作为第一条消息发送给 WorkBuddy。

---

## 模板1：主知识库（主 WorkBuddy）

```
新对话开始。

请按以下顺序读取文件，了解上下文：

1. 读取 `SOUL.md` — 了解你的身份和行为准则
2. 读取 `IDENTITY.md` — 确认你的名字和角色
3. 读取 `USER.md` — 了解用户是谁
4. 读取 `workbuddy-wiki-schema.md` — 了解 Wiki 方法和约定
5. 读取 `wiki/log/index.md` — 了解近期操作
6. 读取 `wiki/log/[今天日期].md` — 了解今天的工作进度（如果文件存在）
7. 读取 `.workbuddy/memory/MEMORY.md` — 获取跨会话重要事实

读取完成后，简要汇报：
- 当前 Wiki 规模（从 `wiki/index.md`）
- 昨天做了什么（从 `wiki/log/`）
- 今天继续做什么
```

---

## 模板2：子知识库（新 WorkBuddy 实例）

```
新对话开始。

请按以下顺序读取文件，了解上下文：

1. 读取 `SOUL.md` — 了解你的身份和行为准则
2. 读取 `IDENTITY.md` — 确认你的名字和角色
3. 读取 `USER.md` — 了解用户是谁
4. 读取 `workbuddy-wiki-schema.md` — 了解 Wiki 方法和约定
5. 读取 `wiki/log/index.md` — 了解近期操作
6. 读取 `wiki/log/[今天日期].md` — 了解今天的工作进度（如果文件存在）
7. 读取 `.workbuddy/memory/MEMORY.md` — 获取跨会话重要事实
8. 读取 `wiki/统筹/跨知识库协作指南.md` — 了解跨知识库协作流程

读取完成后，简要汇报：
- 当前知识库规模（从 `wiki/index.md`）
- 昨天做了什么（从 `wiki/log/`）
- 今天继续做什么
- 如何与主知识库协作（从 `wiki/统筹/跨知识库协作指南.md`）
```

---

## 模板3：简短版（当对话历史快满时）

```
继续工作。

请快速读取：
1. `wiki/log/index.md` — 最后一条日志日期
2. `wiki/log/[最后日期].md` — 了解上次进度
3. `workbuddy-wiki-schema.md` — 确认当前遵循的约定

然后告诉我：上次做到哪里，接下来做什么。
```

---

## 使用建议

1. **首次开启新对话**：使用**模板1**或**模板2**
2. **对话历史快满时**：使用**模板3**（简短版）
3. **每次会话结束前**：确保重要信息已写入 `wiki/log/[今天日期].md`
