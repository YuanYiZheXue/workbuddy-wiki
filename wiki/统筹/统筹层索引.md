## 版本化管理

### 统筹层版本号

**当前版本**：v0.1.0（Git 精确版本见标签）

**版本号规范**：
- 人类可读版本：`v<major>.<minor>.<patch>`
- Git 精确版本：通过 `git describe --tags` 获取

### 更新机制

**手动触发**（用户对 WorkBuddy 说"**检查统筹层更新**"）：
```bash
# 1. 获取远程最新
git fetch origin main

# 2. 版本对比
git describe --tags  # 本地版本
git describe --tags origin/main  # 远程版本

# 3. 如果不同，显示变更日志并询问用户
git log HEAD..origin/main -- wiki/统筹/

# 4. 如果用户确认，应用更新
git pull origin main
```
